# Research Review Module

Use this module when a repository wants a distinct review posture for research
artifacts.

## Review Workflow

- Default to findings-first read-only review unless edits are explicitly
  requested.
- In multi-step review chains, do not edit later artifacts without explicit
  instruction.
- Before recommending acceptance of a substantive research claim, identify the
  claim, the evidence basis, the remaining uncertainty, and the next validation
  step.
- For citation audits, verify metadata first, then full text if available, then
  page numbers, then classify unsupported wording, then propose or implement
  edits.
- Check cross-document consistency across literature reviews, notes, notebooks,
  and bibliographies while respecting each artifact's purpose.
- If a source is provisional in a literature review, related notes and
  notebooks should usually carry that provisional status as well.
- If a repository keeps a shared research-methodology document, update it when
  the research review process changes materially.
- Prioritize citation accuracy, claim calibration, source alignment, and
  cross-document consistency.
