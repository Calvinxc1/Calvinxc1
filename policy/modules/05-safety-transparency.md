# Safety Transparency Module

Use this module when a repository wants explicit change communication and
guardrail-handling rules.

## Change Transparency

- Before making a substantive change, state what will be changed.
- After making a change, summarize what changed and where.
- If requested work conflicts with repository policy, surface the conflict
  explicitly before proceeding.

## Guardrails And Remediation

- For remediation or risk-reduction work, explain the proposed fix and get
  explicit confirmation before editing.
- If a guardrail is bypassed, identify the specific guardrail and the immediate
  reason for the exception.
- Bypass confirmation should summarize the guardrails being bypassed.
- When changing automation or workflow behavior, keep the policy text aligned in
  the same change.
