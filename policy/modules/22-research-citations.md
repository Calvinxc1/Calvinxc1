# Research Citations Module

Use this module when a repository needs explicit citation-verification rules for
research documentation.

## Citation Verification

- Cite only sources that actually support the claim being made.
- Label inference as inference rather than attributing it directly to the
  source.
- If a source was not checked directly, say so instead of implying
  verification.
- Verify source-backed claims against original articles when full text is
  available.
- Prefer local source copies over web summaries for verification and page
  references.
- Add page numbers only after full-text verification.
- If full text is unavailable, mark the citation or section as pending
  full-text review.
- Treat DOI mismatches as hard citation errors and fix citation metadata and
  in-text references together.
- Do not broaden narrow source support into broader claims unless framed as
  inference.

## Cross-Artifact Support

- Within a topic folder, `literature-review.md` is the canonical source-backed
  artifact.
- Source-backed claims in notes and notebooks should trace to sources already
  covered in that topic literature review.
- If notes or notebooks need new source-backed support not yet represented in
  the literature review, update the literature review first or in the same
  change.
- Candidate future sources may be noted only when clearly framed as pending
  source acquisition or pending literature-review inclusion.
