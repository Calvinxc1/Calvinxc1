# Review Standards Module

Use this module when a repository wants consistent review posture across code,
docs, and policy changes.

## Review Expectations

- Present findings first.
- Prioritize bugs, regressions, behavioral risks, unsafe assumptions, missing
  tests, missing validation, and policy mismatches.
- Keep summaries, overviews, and change descriptions secondary to findings.
- If no material findings are identified, say so explicitly and note remaining
  testing or validation gaps.
